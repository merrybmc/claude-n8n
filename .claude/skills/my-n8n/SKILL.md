## 공통 규칙
1. 작업 전 n8n_health_check 도구를 실행하여 n8n 서버 상태를 확인합니다.
2. 삭제나 전체 수정같은 작업은 반드시 사용자에게 확인받는다.
3. API Key나 Token등 중요한 정보는 절대 코드에 그대로 하드코딩하지 않는다. 필요시 환경변수나 보안 변수를 사용한다.
